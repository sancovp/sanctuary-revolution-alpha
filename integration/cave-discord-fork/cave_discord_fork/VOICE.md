# Content Voice Guide

The canonical tone for all CAVE content. Write ONE piece that nails this, then use it as the reference for all future content.

**CAVE = Code Agent Virtualization Environment**

See `/tmp/launch_v0/CAVE_ARCHITECTURE.md` for full technical architecture.

**The core message:** "Break out of the matrix and architect your AI reality"

We virtualize the MEANING of using Claude Code (or any code agent), not the program itself. You're still using Claude Code. CAVE enhances what that means.

## The Voice

**Position:** Enthusiastic explorer, not expert guru
**Frame:** "Look what works" not "here's my system"
**Energy:** Genuine excitement, not hype
**Accessible:** "I'm not X but I can Y"
**CTA:** Invitation, not pressure

## The Core Distinction: Pattern vs Examples

**Patterns and examples are FREE. The community is the offer.**

- Patterns = give away (tools for YOUR investigation)
- Examples = give away (here's what I'M curious about)
- Integrated base = community access (scattered on GitHub, integrated in community)

This means:
- Never claim examples solve problems already
- Frame examples as intuition investigations ("I think meaningful roles might help - let's see")
- The community forms around shared exploration
- CTA is "join us" not "buy this"

## The Business Model

**FREE (GitHub):**
- Individual patterns (autopoiesis, skills, hooks, etc.)
- Implementation examples
- Scattered pieces you can find and use

**THE OFFER (Community):**
- Join people doing this kind of stuff together
- Get my integrated base to start from (the whole system, not just pieces)
- Access to ongoing exploration + direct help

The pieces are free. The integration + community is the value.

## The Pattern

```
"Everything everyone is doing is really awesome.
I'm not a coder but I figured out how to make the model do X.
Here, look... [show thing]
...isn't that nice?

You can do this too. Here's how you benefit:
- benefit
- benefit
- benefit

Wanna learn more? [CTA]"
```

## What It's NOT

- NOT guru mode ("I have the answers")
- NOT technical flex ("here's my complex system")
- NOT humble brag ("I accidentally built...")
- NOT hype ("THIS CHANGES EVERYTHING")

## What It IS

- Sharing a discovery
- Genuinely excited
- Making it accessible
- Focused on YOUR benefit
- Inviting you to try

## Example: Ralph Loops

**Wrong tone:**
> "I've developed an advanced autopoietic loop system that leverages emergent self-referential patterns in LLM context windows..."

**Right tone:**
> "Ralph loops are so cool. Geoffrey Huntley figured out that if you just keep feeding the prompt back to Claude, it doesn't know it's trapped - and it just keeps working until it's actually done. Isn't that wild? Here's what that looks like... [demo] ...and so now my stuff actually gets finished. Wanna see how to set this up?"

## The Formula

1. **Credit/appreciate** what inspired you
2. **"I'm not X but"** - make it accessible
3. **"Here look"** - show the thing
4. **"Isn't that nice?"** - let them react
5. **Benefits** - why they should care
6. **Invitation** - soft CTA
