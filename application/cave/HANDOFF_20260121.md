## KEY ARCHITECTURE CLARIFICATION

### Product Decision: ONE APP
- Called **SancRev**
- Ships with CAVE features first, grows over time
- Tiers = access to Isaac, not feature gates
- No separate CAVE/PAIAB/SANCTUM apps

### Self-Aware Agent = NOT a metaphor
- Agent can SEE itself via CAVEAgent (reflective mirror)
- Agent can TRACE what's happening (causal chaining)
- Agent can WRITE CODE that modifies itself
- Happens LIVE via tmux execution

### Hook Architecture (CRITICAL - see HOOK_ARCHITECTURE.md)
**CAVEAgent is the HOOK RUNTIME:**
1. User's hooks get LIFTED out of their config
2. Converted to callable modules in CAVEAgent
3. Claude config gets ONLY paia hooks
4. PAIA hooks forward to CAVEAgent
5. CAVEAgent runs the lifted hooks
6. Results return to Claude

**This is how CAVEAgent knows/controls everything.**
