## KEY INSIGHT: AgentInferenceLoop

**A loop is NOT just hook handlers. It's a complete autonomous execution pattern:**

```python
class AgentInferenceLoop:
    name: str
    hooks_enabled: Dict[str, bool]  # which hooks to turn on
    handlers: Dict[str, Callable]   # what they do when fired
    conditions: Dict[str, Callable] # when to transition/stop/continue
    on_start: Callable              # runs when loop loads
    on_stop: Callable               # runs when loop unloads
    tmux_messages: List[str]        # can inject to agent if coded that way
```

**CAVEAgent.load_loop(config):**
1. Validate config
2. Enable hooks → writes `/tmp/hook_config.json`
3. Register handlers with HookRouterMixin
4. Run `on_start()`
5. Loop is HOT - events flow through handlers
6. Conditions checked, tmux hit if needed
7. Until conditions trigger `on_stop()`

**The genius:** Agent can load loop ONTO ITSELF. Then operates under that loop.
Conditions like "if autocompact → persist state, inject on resume".

**CAVEAgent = state closure. MainCodeAgent pushes events. Loops define behavior.**
