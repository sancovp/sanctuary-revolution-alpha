# Discord Concept for CAVE

## Channel Structure (10 channels)

```
WELCOME/
└── #welcome (about info, onboarding)

PAIAB/
├── #paiab-overview
├── #paiab-journeys (public - obstacle→solution stories = ADS)
└── #paiab-frameworks (private - extracted knowledge)

SANCTUM/
├── #sanctum-overview
├── #sanctum-journeys (public - obstacle→solution stories = ADS)
└── #sanctum-frameworks (private - extracted knowledge)

CAVE/
├── #cave-overview
├── #cave-journeys (public - obstacle→solution stories = ADS)
└── #cave-frameworks (private - extracted knowledge)
```

## Content Flow

```
Work → Extract Frameworks (private) → Demonstrate via Journeys (public/ads) → Pull into value ladder
```

## Content Templates

### Blog Template
- Title
- Hook
- Body (uses identity constants)
- CTA

### LinkedIn Post Template
- Hook
- Body
- CTA
- Hashtags

### Tweet Template
- Hook (max 280 chars)
- Thread (optional)

### YouTube Video Model
```
├── thumbnail
├── title
└── script
    ├── intro
    ├── who_am_i (CONSTANT - set once)
    ├── body
    ├── insights
    ├── cta (CONSTANT - set once)
    └── demo (optional, embedded in body)
```

## Identity Constants (set once, used everywhere)

- `who_am_i` - Used in YouTube, Blog about, introductions
- `cta` - Call to action used across all content
- `twitter_bio` - Twitter/X profile bio
- `linkedin_bio` - LinkedIn profile bio
- `about_short` - One-liner description
- `about_long` - Full about section

## Journey = Ad Structure

A Journey post documents:
1. The **obstacle** faced
2. The **solution** discovered
3. The **transformation** achieved
4. The **framework** extracted (links to private frameworks channel)

Journeys are public content that demonstrate expertise and pull people into the value ladder.

## Framework Structure

Frameworks are extracted knowledge:
1. **Problem pattern** - What recurring problem does this solve?
2. **Solution pattern** - What's the repeatable solution?
3. **Implementation** - How to apply it?
4. **Source journey** - Which journey demonstrated this?
