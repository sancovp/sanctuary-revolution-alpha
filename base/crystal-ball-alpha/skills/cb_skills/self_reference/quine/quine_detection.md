# Quine Detection

> Recognize when a structure describes itself — the fixed point of the ontology

## When to use
You've built a template and its solution space, and you notice that the solution space CONTAINS the template as a node. This is the quine.

## The pattern

1. **TrajectoryTemplate** describes how to climb the abstraction ladder:
   `Concrete → Compose → Fold → Extract → Abstract → Meta → Govern → Self-Apply`

2. **TemplateSolutionSpace** describes how to SOLVE the template:
   `Domain_Seed → Pillar_Discovery → Depth_Generation → ..→ Quine_Detection`

3. **The quine**: `TemplateSolutionSpace.Quine_Detection` contains `TrajectoryTemplate` as a child node.
   - Coordinate: `root.6.0` = `7.1`
   - The solution space's 7th step (detect quines) has the template as its child
   - Meaning: "the quine you detect is the thing you used to get here"

## Composed coordinates
```
TrajectoryTemplate → scry all    → "12345678"
TemplateSolutionSpace → scry all → "1234567.1"

The ".1" at the end of the solution space IS the template.
The template's "8" (Self_Application) IS the instruction to create the solution space.
```

## Why this matters
This is a **fixed point in ontological space**. The structure describes itself.
It means any agent that follows the template will eventually arrive at the template itself.
The process of understanding IS the thing being understood.

This is not a bug. This is the Christopher Alexander "Quality Without a Name" —
the structure that naturally emerges when you follow the pattern language is the pattern language itself.
