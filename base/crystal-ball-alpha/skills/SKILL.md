# Crystal Ball Skill

Here are the categories of what to do with Crystal Ball. Navigate appropriately.

## Categories

| Domain | What it covers |
|--------|---------------|
| `space_creation` | Building individual spaces from scratch — pillars, depth, attributes |
| `composition` | Folding spaces into groups, groups into empires — Order N systems |
| `abstraction` | Climbing the ladder: instance → solution space → category → meta → governance |
| `querying` | Scry (superposition queries), bloom (interior navigation), resolve |
| `attribute_design` | Defining spectra, mining quality dimensions, cross-level attributes |
| `self_reference` | Template extraction, quine detection, the pattern describing itself |

## Quick Reference

```
# List all skill processes:
bash .agent/skills/crystal-ball/scripts/tree.sh

# The MCP tool:
crystal_ball(cmd, body)

# Core commands:
create_space, add_point, add_attribute, bloom, scry, resolve, get_space, list_spaces
```

## Usage Pattern

1. **Start concrete** — pick a domain, create a space, add pillars
2. **Add depth** — sub-nodes under pillars, sub-sub-nodes under those
3. **Add attributes** — define spectra for quality dimensions
4. **Compose** — group spaces into higher-order spaces
5. **Fold** — repeat composition to create Order N systems
6. **Abstract** — extract the solution space (the pattern behind the instance)
7. **Self-apply** — the template describes how to build templates (quine)

Each `cb_skills/{domain}/{subdomain}/{process}.md` is a standalone recipe.
