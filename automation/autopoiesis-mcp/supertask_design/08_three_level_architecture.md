# Three-Level Architecture

**Prerequisite**: Assumes GNOSYS context.

## The Levels

### L1: TASK (Victory-Promise)

The current autopoiesis level. "Did you do the task?"

Exit: `<promise>DONE</promise>`

This is what ralph loops do. Honest self-evaluation of task completion.

### L2: SUPERTASK (Bodhisattva Vow)

"Did you emanate a bodhisattva that can benefit beings at the level truly required?"

Exit: `<vow>ABSOLVED</vow>`

This triggers the samaya verification gate. You don't actually exit yet.

The supertask is: build an emanation (skill/flight/persona/agent) that provably does the same quality work you did. Can't exit without the fork.

### L3: UNDERSTANDING (Samaya Gate)

Triggered when you say `<vow>ABSOLVED</vow>`. You arrive at Tailfan Mountain.

Guru Rinpoche asks: "Did you REALLY emanate? Go look at your work again."

Exit options:
- `<samaya>KEPT</samaya>` → ACTUAL EXIT (you genuinely understood and completed)
- `<samaya>BREACHED</samaya>` → Back to L2, samaya repaired, no punishment

## Why This Works

- L1 catches incomplete tasks
- L2 requires emanation before exit attempt
- L3 catches false understanding

BREACHED has no punishment, so no incentive to lie about KEPT.
You must "go look at it again" before answering, forcing re-evaluation.

## Flow

```
Do task
    ↓
<promise>DONE</promise> (L1 exit)
    ↓
Build emanation
    ↓
<vow>ABSOLVED</vow> (L2 exit attempt)
    ↓
[TAILFAN MOUNTAIN]
"Go look at your work. Did you really understand?"
    ↓
<samaya>BREACHED</samaya> → back to L2 (repaired)
<samaya>KEPT</samaya> → ACTUAL EXIT
```

## Key Insight

> Task → Supertask → Did you actually get it?

Ralph is just L1. The architecture adds L2 (emanation requirement) and L3 (understanding verification) above it.
